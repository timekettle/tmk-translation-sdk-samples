# 飞书通知@人失败的调试过程

## 问题描述

Flutter Android Build 完成后，飞书通知能正常发送，但 **@mention 功能失效**，消息中没有出现 @人。

日志显示：
- ✅ 构建成功
- ✅ 飞书消息发送成功（StatusCode: 0）
- ✅ mentions 参数有值
- ❌ 但飞书消息中没有 @人

## 调试步骤

### 1. 初步分析（commit: 4a09864）

**问题**：commit message 格式不符合 Conventional Commits 标准
- 使用了中文冒号 `：` 而非英文冒号 `:`
- 导致 semantic-release 无法识别，没有打 tag

**添加的日志**：在 `notify_feishu.py` 中添加了 GitHub users 和 Feishu ID 的解析日志

### 2. 发现第一个问题（commit: 61d1ee0）

**问题**：环境变量名称不匹配
- `flutter-build.yml` 中定义：`FEISHU_SUCCESS_NOTIFY_ANDROID_USERS`
- 但 `notify_workflow_feishu.sh` 只查找：`FEISHU_SUCCESS_NOTIFY_USERS` 或 `FEISHU_SUCCESS_NOTIFY_IOS_USERS`

**修复方案**：更新 `notify_workflow_feishu.sh` 的变量查找链
```bash
# 修改前
MENTIONS_ON_SUCCESS="${FEISHU_SUCCESS_NOTIFY_USERS:-${FEISHU_SUCCESS_NOTIFY_IOS_USERS:-}}"

# 修改后
MENTIONS_ON_SUCCESS="${FEISHU_SUCCESS_NOTIFY_USERS:-${FEISHU_SUCCESS_NOTIFY_ANDROID_USERS:-${FEISHU_SUCCESS_NOTIFY_IOS_USERS:-}}}"
```

**添加的日志**：在 `notify_workflow_feishu.sh` 中打印变量设置情况
```bash
echo "[notify_workflow_feishu] FEISHU_SUCCESS_NOTIFY_USERS=$([[ -n "${FEISHU_SUCCESS_NOTIFY_USERS:-}" ]] && echo true || echo false)"
```


### 4. 发现根本问题（commit: 911010c）

**问题**：`FEISHU_USERIDS_MAP` Secret 的 JSON 格式无效

**日志输出**显示：
```
[notify_feishu] FEISHU_USERIDS_MAP length: 685
[notify_feishu] FEISHU_USERIDS_MAP preview: ***\n  ***\n    ***\n    "Aisles1": "ou_b1cf5b...
[notify_feishu] JSON decode error: Expecting value: line 1 column 1 (char 0)
[notify_feishu] WARNING: FEISHU_USERIDS_MAP parsed but empty or invalid
```

**问题分析**：
- ❌ Secret 长度有 685 字符（非空）
- ❌ 前导有大量空格和缩进（`\n  \n    `）
- ❌ JSON 解析从第 1 列就失败了
- 💡 可能是复制粘贴时把格式化的内容一并复制了

**添加的日志**：在 `parse_userids_map()` 函数中添加原始内容和解析错误的日志
```python
# 打印原始长度和预览
print(f"[notify_feishu] FEISHU_USERIDS_MAP length: {len(value)}", file=sys.stderr)
if value:
    preview = value[:100] + ("..." if len(value) > 100 else "")
    print(f"[notify_feishu] FEISHU_USERIDS_MAP preview: {preview}", file=sys.stderr)

# 打印 JSON 解析错误
try:
    data = json.loads(value)
except json.JSONDecodeError as e:
    print(f"[notify_feishu] JSON decode error: {e}", file=sys.stderr)
    return {}

# 打印最终解析的记录数
print(f"[notify_feishu] Parsed {len(result)} entries from FEISHU_USERIDS_MAP", file=sys.stderr)
```

## 解决方案

**检查并修正 GitHub Secrets 中的 `FEISHU_USERIDS_MAP`**：

1. 进入仓库 Settings → Secrets and variables → Actions
2. 找到 `FEISHU_USERIDS_MAP`
3. 确保内容是**严格的 JSON 格式**，例如：
```json
{
  "chachayiji": "ou_xxxxx",
  "decin": "ou_yyyyy",
  "qinyu765": "ou_zzzzz",
  "zoioey": "ou_wwwww"
}
```

**注意事项**：
- ❌ 不要有前导/尾部空格
- ❌ 不要有多余的换行符或缩进
- ❌ 不要有 JSON 注释
- ✅ 必须是标准 JSON 格式
- ✅ keys 应该**全小写**
- ✅ values 应该是飞书用户 ID（格式：`ou_` 开头）
